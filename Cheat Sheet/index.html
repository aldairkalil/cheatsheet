<form>

The HTML <form> element is used to collect and send information to an external source.


<textarea>

The textarea element is used when creating a text-box for multi-line input (e.g. a comment section). The element supports the rows and cols attributes which determine the height and width, respectively, of the element.


<input>

The HTML <input> element is used to render a variety of input fields on a webpage including text fields, checkboxes, buttons, etc. <input> element have a type attribute that determines how it gets rendered to a page.


<select>

The HTML <select> element can be used to create a dropdown list. A list of choices for the dropdown list can be created using one or more <option> elements. By default, only one <option> can be selected at a time.


<datalist>

When using an HTML input, a basic search/autocomplete functionality can be achieved by pairing an <input> with a <datalist>. To pair a <input> with a <datalist> the <datalist>'s list value must match the value of the id of the <datalist>. The datalist element is used to store a list of <otions>.


<label>

The HTML <label> element provides identification for a specific <input> based on matching values of the <input>‘s id attribute and the <label>‘s for attribute. By default, clicking on the <label> will focus the field of the related <input>.


Input Types

<input> element is used to render input fields including text fields, checkboxes, buttons, etc. <input> element has type attributes that determine how it's rendered to a page.

  • checkbox: When using an HTML input element, the type="checkbox" attribute will render a single checkbox item. To create a group of checkboxes related to the same topic, they should all use the same name attribute.
  • number: HTML input elements can be of type number. These input fields allow the user to enter only numbers and a few special characters inside the field.
  • range: The range slider will act as a selector between a minimum and a maximum value.
  • text: This renders a single row input field that users can type text inside.
  • radio: Multiple radio buttons of a related topic are given the same name attribute value.
  • password: The HTML <input> element can have the attribute type="password" that renders a single row input field which allows the user to type censored text inside the field.
  • submittable: HTML <input> elements can have a type attribute set to submit, by adding type="submit". With this attribute included, a submit button will be rendered and, by default, will submit the <form> and execute its action.

Other Attributes

  • required: In HTML, input fields have an attribute called required which specifies that the field must include a value.
  • max: HTML <input>'s of type number have an attribute called max that specifies the maximum value for the input field.
  • maxlength: In HTML, input fields with type text have an attribute called maxlength that specifies the maximum number of characters that can be entered into the field.
  • pattern: In a text input element, the pattern attribute uses a regular expression to match against (or validate) the value of the <input>, when the form is submitted.
  • minlength: In HTML, an input field of type text has an attribute that supports minimum length validation.
  • min: In HTML, input fields with type number have an attribute called min that specifies the minimum value that can be entered into the field.
  • name: In order for a form to send data, it needs to be able to put it into key-value pairs. This is achieved by setting the name attribute of the input element.
